# Documentação do Módulo CRM Tasks
Este documento descreve como funciona o módulo de tarefas do CRM e como integrá-lo com o frontend.

## Estrutura de Dados

### Tipos de Tarefa
- EMAIL: Tarefa de envio de e-mail
- CALL: Tarefa de ligação
- MEETING: Tarefa de reunião
- WHATSAPP: Tarefa de mensagem via WhatsApp
- FOLLOW_UP: Tarefa de acompanhamento
- ONBOARDING: Tarefa de onboarding
- IN_PERSON_VISIT: Tarefa de visita presencial
- TASK: Tarefa genérica

### Status de Tarefa
- PENDING: Pendente
- IN_PROGRESS: Em andamento
- COMPLETED: Concluída
- CANCELED: Cancelada
- LATE: Atrasada
- SCHEDULED: Agendada

## Endpoints

### 1. Criar Tarefa
```bash
curl -X POST http://localhost:3000/crm/tasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer seu_token_jwt" \
  -d '{
    "title": "Título da Tarefa",
    "description": "Descrição da tarefa",
    "type": "MEETING",
    "dueDate": "2024-03-20T10:00:00Z",
    "status": "PENDING",
    "ownerId": "uuid_do_owner",
    "userIds": ["uuid_usuario1", "uuid_usuario2"],
    "negotiationId": "uuid_negociacao",
    "companyId": "uuid_empresa",
    "contactId": "uuid_contato",
    "recurrence": {
      "type": "WEEKLY",
      "weekDays": [1, 3, 5],
      "hour": 10,
      "minute": 0
    }
  }'
```

### 2. Listar Tarefas
```bash
curl -X GET "http://localhost:3000/crm/tasks?page=1&limit=10&status=PENDING&type=MEETING" \
  -H "Authorization: Bearer seu_token_jwt"
```

### 3. Buscar Tarefa por ID
```bash
curl -X GET http://localhost:3000/crm/tasks/uuid_da_tarefa \
  -H "Authorization: Bearer seu_token_jwt"
```

### 4. Atualizar Tarefa
```bash
curl -X PATCH http://localhost:3000/crm/tasks/uuid_da_tarefa \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer seu_token_jwt" \
  -d '{
    "title": "Novo Título",
    "status": "IN_PROGRESS",
    "recurrence": {
      "type": "MONTHLY",
      "monthDays": [1, 15],
      "hour": 14,
      "minute": 30
    }
  }'
```

### 5. Remover Tarefa
```bash
curl -X DELETE http://localhost:3000/crm/tasks/uuid_da_tarefa \
  -H "Authorization: Bearer seu_token_jwt"
```

## Recorrência de Tarefas

A recorrência é armazenada em uma tabela separada e vinculada à tarefa principal. Isso permite uma melhor organização e gerenciamento das tarefas recorrentes.

### Tipos de Recorrência

1. **Semanal (WEEKLY)**
   - Define os dias da semana em que a tarefa deve ocorrer
   - Exemplo: Toda segunda, quarta e sexta às 10:00
   ```json
   {
     "type": "WEEKLY",
     "weekDays": [1, 3, 5],
     "hour": 10,
     "minute": 0
   }
   ```

2. **Quinzenal (BIWEEKLY)**
   - Define os dias da semana em que a tarefa deve ocorrer a cada duas semanas
   - Exemplo: Toda segunda-feira às 14:30, a cada duas semanas
   ```json
   {
     "type": "BIWEEKLY",
     "weekDays": [1],
     "hour": 14,
     "minute": 30
   }
   ```

3. **Mensal (MONTHLY)**
   - Define os dias do mês em que a tarefa deve ocorrer
   - Exemplo: Todo dia 1 e 15 de cada mês às 09:00
   ```json
   {
     "type": "MONTHLY",
     "monthDays": [1, 15],
     "hour": 9,
     "minute": 0
   }
   ```

### Campos da Recorrência

- `type`: Tipo de recorrência (WEEKLY, BIWEEKLY, MONTHLY)
- `weekDays`: Array com os dias da semana (0 = Domingo, 1 = Segunda, ..., 6 = Sábado)
- `monthDays`: Array com os dias do mês (1-31)
- `hour`: Hora do dia (0-23)
- `minute`: Minuto da hora (0-59)

## Dicas para Implementação no Frontend

1. **Formulário de Criação/Edição**
   - Campos obrigatórios: título, tipo, data de vencimento, status, owner
   - Campos opcionais: descrição, usuários auxiliares, negociação, empresa, contato, recorrência
   - Para recorrência, mostrar campos específicos baseado no tipo selecionado

2. **Listagem de Tarefas**
   - Mostrar informações principais: título, tipo, status, data de vencimento
   - Indicador visual para tarefas recorrentes
   - Filtros por tipo, status, data, owner, etc.

3. **Visualização de Tarefa**
   - Mostrar todos os detalhes da tarefa
   - Se for recorrente, mostrar a configuração de recorrência
   - Opções para editar, marcar como concluída, cancelar, etc.

4. **Tratamento de Recorrência**
   - Validar os dias da semana/mês conforme o tipo de recorrência
   - Garantir que hora e minuto sejam válidos
   - Mostrar preview das próximas ocorrências

5. **Design Responsivo**
   - Adaptar layout para diferentes tamanhos de tela
   - Priorizar informações mais importantes em telas menores

## Exemplos de Payloads

### 1. Tarefa Simples
```json
{
  "title": "Ligar para cliente",
  "description": "Confirmar agendamento da reunião",
  "type": "CALL",
  "dueDate": "2024-03-20T14:30:00Z",
  "status": "PENDING",
  "ownerId": "uuid_do_owner",
  "contactId": "uuid_do_contato"
}
```

### 2. Tarefa com Recorrência Semanal
```json
{
  "title": "Reunião de equipe",
  "description": "Reunião semanal de alinhamento",
  "type": "MEETING",
  "dueDate": "2024-03-20T10:00:00Z",
  "status": "SCHEDULED",
  "ownerId": "uuid_do_owner",
  "userIds": ["uuid_usuario1", "uuid_usuario2"],
  "recurrence": {
    "type": "WEEKLY",
    "weekDays": [1],
    "hour": 10,
    "minute": 0
  }
}
```

### 3. Tarefa com Recorrência Mensal
```json
{
  "title": "Relatório mensal",
  "description": "Enviar relatório de atividades",
  "type": "EMAIL",
  "dueDate": "2024-03-01T09:00:00Z",
  "status": "SCHEDULED",
  "ownerId": "uuid_do_owner",
  "recurrence": {
    "type": "MONTHLY",
    "monthDays": [1],
    "hour": 9,
    "minute": 0
  }
}
```

## Observações Importantes

1. **Datas**
   - Todas as datas devem ser enviadas no formato ISO 8601
   - O campo `dueDate` é obrigatório e define a data de vencimento da tarefa

2. **UUIDs**
   - Todos os IDs (owner, users, negotiation, company, contact) devem ser UUIDs válidos
   - Verificar se os IDs existem antes de criar/atualizar a tarefa

3. **Recorrência**
   - A recorrência é opcional
   - Se fornecida, todos os campos necessários para o tipo selecionado devem ser preenchidos
   - A data de vencimento (`dueDate`) serve como data inicial para a recorrência
